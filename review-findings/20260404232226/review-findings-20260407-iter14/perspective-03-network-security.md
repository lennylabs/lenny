| 2 | NET-035 | Medium | **Gateway ingress NetworkPolicy in `lenny-system` missing port 8443 (LLM proxy) from agent namespaces.** The agent-namespace `allow-pod-egress-llm-proxy` policy (lines 5470–5492) permits egress from proxy-mode pods (`lenny.dev/delivery-mode: proxy`) to the gateway on port 8443. However, the `lenny-system` component NetworkPolicy table (line 5525) lists the gateway's ingress as: "External ingress (TCP 443 from Ingress controller namespace); agent namespaces (TCP, gateway gRPC port for pod-to-gateway traffic)." The "gateway gRPC port" is 50051 — port 8443 is not mentioned. Under the `lenny-system` `default-deny-all` policy (lines 5510–5519), the gateway's ingress rule must explicitly allow port 8443 from agent namespaces for LLM proxy traffic to flow. Without this, all proxy-mode LLM requests would be silently dropped at the `lenny-system` boundary despite the agent-side egress being open. **Fix:** Add "agent namespaces (TCP 8443, LLM proxy — pods with `lenny.dev/delivery-mode: proxy` only)" to the gateway's Ingress Allowed column. | 13.2 | 5525 |
| 3 | NET-036 | Medium | **MinIO missing from `lenny-system` component NetworkPolicy table (self-managed profile).** The self-managed deployment profile (Section 17.9, line 8374) deploys MinIO as a Kubernetes workload at `minio.lenny-system:9000`, placing it inside the `lenny-system` namespace. The gateway's egress allows MinIO (TCP 9443, TLS). However, MinIO has no row in the `lenny-system` component NetworkPolicy table (lines 5523–5530). PgBouncer correctly has its own row with the "(self-managed profile only)" caveat specifying both egress and ingress rules. MinIO needs the same treatment: ingress from gateway pods (TCP 9443) and egress to `kube-system` CoreDNS (UDP/TCP 53) at minimum. ## Verification notes

Checked the following areas for issues; all were internally consistent:

- **Three base NetworkPolicy manifests** (lines 5388–5492): `default-deny-all`, `allow-gateway-ingress`, `allow-pod-egress-base`, and supplemental `allow-pod-egress-llm-proxy` are structurally correct. Pod selectors use `lenny.dev/managed: "true"`, namespace selectors use the immutable `kubernetes.io/metadata.name` label (not spoofable custom labels). Port numbers are consistent with component definitions.
- **Lateral movement between pods in the same pool namespace**: Blocked. `default-deny-all` denies all ingress/egress by default. `allow-pod-egress-base` only opens egress to gateway (port 50051) and CoreDNS (port 53) in `lenny-system`. No pod-to-pod ingress or egress rule exists. The `internet` profile's `0.0.0.0/0` carries `except` clauses for cluster pod/service CIDRs (NET-002), with continuous drift detection (NET-022) and preflight validation. Sound.
- **Dedicated CoreDNS for DNS exfiltration**: Operationally practical. HA covered (2+ replicas, PDB `minAvailable: 1`, anti-affinity), full Corefile reference provided with rate limiting (10 qps/pod), response filtering (TXT size threshold, tunneling record types), query logging. `dnsPolicy: None` on agent pods (K8S-033) ensures `/etc/resolv.conf` points to the dedicated instance. Opt-out path (`dnsPolicy: cluster-default`) documented with security trade-off warning. Alerts (`DedicatedDNSUnavailable`, `DedicatedDNSDegraded`) cover failure modes.
- **Network paths bypassing the gateway**: None found. Callback worker (line 5797) runs inside the gateway process with its own goroutine pool and `http.Client`; the NetworkPolicy for callback traffic routes through the gateway's own egress rules (external HTTPS with IMDS exclusions). `provider-direct` egress opens pod-to-LLM-provider CIDRs but is mutually exclusive with `deliveryMode: proxy` (NET-006) and requires sandboxed isolation. All delegation goes through gateway (explicit non-goal: direct pod-to-pod communication).
- **mTLS PKI without service mesh** (Section 10.3): Sufficient. cert-manager with cluster-internal CA, per-component certificate lifecycle (4h for pods, 24h for infrastructure), SPIFFE URIs for pod identity, mutual SAN validation between gateway and Token Service, certificate deny list via Redis pub/sub, proactive idle-pod cert drain, CA rotation procedure with overlap window. Short-lived certs (4h) substitute for CRL/OCSP. `CertExpiryImminent` alert covers cert-manager failures.
- **IMDS blocking**: Consistent. Base policy implicitly blocks (allowlist-only, no broad CIDR). Supplemental policies with `0.0.0.0/0` carry explicit `except` clauses for all three IMDS addresses. Gateway egress also excludes IMDS.
- **Label immutability** (NET-003): `lenny-label-immutability` webhook fully specified with creation guard, mutation guard, fail-closed mode, scoping, HA, and Helm template path.
- **CIDR drift detection** (NET-022): Continuous drift check every 5 minutes, `NetworkPolicyCIDRDrift` critical alert, preflight validation at install/upgrade. Controller does not auto-patch (avoids NetworkPolicy write RBAC). Sound.
- **Redis/PgBouncer TLS enforcement** (NET-004): Server-side controls (`tls-auth-clients yes`, plaintext port disabled for Redis; `client_tls_sslmode = require` for PgBouncer) with gateway startup TLS probe and integration tests. Sound.
- **Admission webhook ingress** (K8S-034 fix from iter13): Admission webhooks row in table (line 5529) correctly specifies ingress from kube-apiserver (TCP 443, CIDR). This was the fix for the K8S-034 high finding in iter13. Verified present.
